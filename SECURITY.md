# 安全策略

感谢你帮助改进项目安全性。

## 支持范围

当前仅对默认分支上的最新代码提供安全修复支持。

## 报告方式

如果你发现了会导致以下情况的安全问题，请不要直接公开提交 Issue：

- 服务器被未授权访问
- 配置或密钥泄露
- 客户端信息被错误导出或暴露
- 证书、私钥、密码生成存在明显缺陷

请通过 GitHub 账户 `renaissance0721` 的私信渠道，或仓库启用后的私密安全报告入口联系维护者。提交时建议包含：

- 问题影响范围
- 复现步骤
- 风险等级判断
- 可选的修复建议

## 处理原则

- 我们会先确认问题是否成立以及影响范围
- 在修复发布前，尽量避免公开可能被直接利用的细节
- 修复完成后，会在合适范围内说明受影响版本和缓解方式
